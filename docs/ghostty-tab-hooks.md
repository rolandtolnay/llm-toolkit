# Ghostty Tab Title & Sound Hooks

Claude Code hooks that auto-rename Ghostty tabs based on conversation topic, show attention emojis when user input is needed, and play Warcraft III sound effects on key events.

All files live under `~/.claude/` (user-global scope). Registered in `~/.claude/settings.json` under `hooks`.

---

## Usage Guide

### Tab Titles

Ghostty tabs auto-rename to a short slug (e.g. `flutter-auth-bug`) based on the conversation topic. Slugs are generated by Haiku on each user message (debounced: 60s cooldown, skips messages under 30 chars, first message always triggers).

### Attention Emojis

| Emoji | Meaning | Trigger |
|-------|---------|---------|
| 🔴 | Blocked — permission prompt or elicitation dialog | `Notification` hook |
| 💬 | Question — Claude asked the user something | `PostToolUse:AskUserQuestion` or `Stop` (heuristic) |

Emojis prepend to the tab title (e.g. `🔴 flutter-auth-bug`). Cleared automatically when the user sends the next message or accepts a plan.

### Sound Effects

Three Warcraft III character classes mapped to three events:

| Event | When | Orc (Peon) | Human (Peasant) | Undead (Acolyte) |
|-------|------|------------|-----------------|------------------|
| `session.start` | Fresh session startup | "Ready" / "What" | "Ready" / "What" | "Ready" / "What" |
| `task.acknowledge` | Tab title changes to new slug | "Yes" variants | "Yes" variants | "Yes" variants |
| `input.required` | Attention emoji added (first time only) | "What" variants | "What" variants | "What" variants |

**Configure per project:**

```sh
peon-class orc       # Use Orc Peon sounds for this project
peon-class human     # Use Human Peasant sounds
peon-class undead    # Use Undead Acolyte sounds
peon-class random    # Random class per sound (default)
peon-class none      # Disable sounds (tab titles still work)
peon-class           # Show current setting
```

This writes `PEON_SOUND_CLASS` to `.claude/settings.local.json` in the current project directory. Claude Code reads env vars from this file at session start.

## Implementation Details

This section is for the LLM maintaining these scripts.

### Architecture Overview

```
settings.json hooks config
    │
    ├─ UserPromptSubmit ──► tabtitle-hook.py ──► Haiku slug ──► set title + task.acknowledge sound
    │
    ├─ Notification ──────► tab-attention-hook.py ──► add 🔴 + input.required sound
    │
    ├─ PostToolUse
    │    ├─ AskUserQuestion ► tab-attention-hook.py ──► add 💬 + input.required sound
    │    └─ ExitPlanMode ───► plan-accept-hook.py ──► clear emoji + task.acknowledge sound
    │
    ├─ Stop ──────────────► tab-stop-question-hook.py ──► Haiku question check ──► add 💬 + input.required sound
    │
    └─ SessionStart ──────► session-sound-hook.py ──► session.start sound
```

### File Map

| File | Location | Purpose |
|------|----------|---------|
| `sound_utils.py` | `~/.claude/hooks/` | Shared module: `play_sound(event)` — reads `PEON_SOUND_CLASS`, picks random file, fires `afplay` non-blocking |
| `tabtitle-hook.py` | `~/.claude/hooks/` | UserPromptSubmit: clears emoji, generates slug via Haiku, sets title, plays task.acknowledge |
| `tab-attention-hook.py` | `~/.claude/hooks/` | Notification + PostToolUse:AskUserQuestion: adds attention emoji, plays input.required |
| `tab-stop-question-hook.py` | `~/.claude/hooks/` | Stop: heuristic question detection via Haiku, adds 💬, plays input.required |
| `plan-accept-hook.py` | `~/.claude/hooks/` | PostToolUse:ExitPlanMode: clears emoji after plan acceptance, plays task.acknowledge |
| `session-sound-hook.py` | `~/.claude/hooks/` | SessionStart: plays session.start sound on fresh startup only |
| `settings.json` | `~/.claude/` | Hook registration (events, matchers, timeouts, async flags) |
| `peon-class()` | `~/.zsh/aliases.zsh` | Shell function: reads/writes `PEON_SOUND_CLASS` in `.claude/settings.local.json` |

### Debounce File Format

Location: `/tmp/claude-tabtitle/{session_id}`

```
line 1: unix timestamp (float) — last rename time, used for 60s cooldown
line 2: title (may include emoji prefix like "🔴 my-slug")
```

Written by `tabtitle-hook.py` after slug generation. Also written by `tab-attention-hook.py` and `tab-stop-question-hook.py` when adding emoji (so subsequent calls detect the emoji is already showing and skip). Read by all hooks to get the current title state.

### Tab Targeting

All hooks use `selected tab of front window` via AppleScript. This means the correct tab must be focused when the title is set.

**Known limitation**: `tabtitle-hook.py` has a ~2s delay for Haiku slug generation. If the user switches tabs during that time, the wrong tab gets renamed. This will be addressed by replacing `claude -p` with a local model for faster slug generation.

**Ghostty AppleScript supports targeted tab addressing** (`first window whose id is "X"` + `tab N of window`) which was verified to work across non-focused tabs and windows. This can be used in a future improvement to capture the tab identity at hook start and use it after the Haiku delay.

### Shared Module: `sound_utils.py`

Single function `play_sound(event)`:
1. Reads `PEON_SOUND_CLASS` env var (default: `random`)
2. `none` → return immediately
3. `random` → pick random class from `(orc, human, undead)`
4. Looks up `~/.claude/sounds/{class}/{event}/`
5. Picks a random file from the directory
6. Fires `subprocess.Popen(["afplay", path])` — non-blocking, fire-and-forget
7. All failures silently caught

### Hook: `tabtitle-hook.py` (UserPromptSubmit)

The most complex hook. Flow:

1. **Recursion guard**: Check `_CLAUDE_HOOK_NESTED` env var, exit if set
2. **Read debounce file**: Get current title
3. **Clear emoji**: Strip attention emoji from title, set clean title immediately
4. **Debounce check**: Skip if within 60s cooldown or message is short (<30 chars). First message always triggers.
5. **Generate slug**: Call `claude -p` with Haiku model (subprocess, 15s timeout). Pass `_CLAUDE_HOOK_NESTED=1` env var to prevent recursive hook execution in the subprocess.
6. **Set title + sound**: If new slug generated, set title and play `task.acknowledge`. Write debounce file with timestamp and slug.

The slug prompt asks Haiku to output a 2-4 word hyphenated slug or `KEEP` if the current title still fits. Validation rejects anything with spaces, special characters, error markers, or over 40 chars.

**Async execution**: Runs with `"async": true` and 30s timeout. This means the user's message is submitted immediately — the tab rename happens in the background.

### Hook: `tab-attention-hook.py` (Notification + PostToolUse:AskUserQuestion)

Registered on two events via two separate entries in `settings.json`:
- `Notification` (no matcher — fires for all notifications, filtered in script)
- `PostToolUse` with matcher `AskUserQuestion`

Flow:
1. Recursion guard
2. Determine emoji: `Notification` with `permission_prompt`/`elicitation_dialog` → 🔴. `PostToolUse` → 💬. Other notification types → exit.
3. Read debounce file — check if emoji is already showing (raw title starts with the target emoji)
4. If already showing → exit (no title change, no sound). This prevents duplicate sounds when Claude requests multiple permissions in succession.
5. Strip any existing emoji, prepend new one, set title, play `input.required`
6. Write debounce file with emoji-prefixed title (so next call detects it's already showing)

### Hook: `tab-stop-question-hook.py` (Stop)

Tier 2 heuristic for detecting when Claude stops with a question that isn't covered by the deterministic signals (AskUserQuestion tool or Notification).

Flow:
1. Recursion guard + `stop_hook_active` check (prevents infinite loops)
2. Read last assistant message from transcript JSONL. **Important**: transcript format nests role inside `entry["message"]["role"]`, not at the top-level `entry["role"]`. Content is at `entry["message"]["content"]`.
3. Pre-filter: skip if no `?` in the last 500 chars (~80% of stops filtered out cheaply)
4. Check debounce file — skip if 💬 emoji already showing
5. Call Haiku via `claude -p` to classify whether the text is asking for user input (YES/NO). Pass `_CLAUDE_HOOK_NESTED=1` to subprocess.
6. If YES: set title with 💬, play `input.required`, update debounce file

### Hook: `plan-accept-hook.py` (PostToolUse:ExitPlanMode)

Handles the edge case where a user accepts a plan: the `Notification` hook added 🔴 when the plan approval dialog was shown, but nothing clears it when the user accepts (since accepting a plan doesn't trigger `UserPromptSubmit`).

Flow:
1. Read debounce file
2. Strip emoji — if no emoji was present, exit (nothing to clear)
3. Set clean title, play `task.acknowledge`
4. Update debounce file

### Hook: `session-sound-hook.py` (SessionStart)

Minimal script. Only plays `session.start` sound on fresh startup (`source == "startup"`). Skips `resume`, `compact`, `clear` events. Has recursion guard since `claude -p` subprocesses also fire `SessionStart`.

### Recursion Guard: `_CLAUDE_HOOK_NESTED`

Two hooks call `claude -p` to invoke Haiku: `tabtitle-hook.py` (slug generation) and `tab-stop-question-hook.py` (question classification). The `claude -p` subprocess fires its own hooks, which would recursively invoke these same scripts, causing:
- Multiple sounds playing (each recursive level plays a sound)
- Slug generation timeouts (recursive `claude -p` calls consume the timeout budget)
- Wrong tab titles (recursive calls fight over the same tab)

Fix: Both `claude -p` calls pass `_CLAUDE_HOOK_NESTED=1` as an environment variable. All 5 hook scripts check this at the top and exit immediately if set. The env var propagates from the parent `claude -p` process to all hooks it fires.

### Sound Deduplication

Sounds are tightly coupled to title changes — they only play when the tab title actually changes:
- `task.acknowledge`: Only in `tabtitle-hook.py` when a new slug is generated, and in `plan-accept-hook.py` when emoji is cleared
- `input.required`: Only in `tab-attention-hook.py` and `tab-stop-question-hook.py` when the emoji state changes (detected by checking the debounce file for existing emoji prefix before setting)

This prevents repeated sounds when Claude requests multiple permissions in quick succession (each triggers `Notification`, but only the first adds the emoji and plays the sound).

### `peon-class` Shell Function

Located in `~/.zsh/aliases.zsh`. Reads/writes `PEON_SOUND_CLASS` in `.claude/settings.local.json` (project-local, gitignored). Uses inline `python3 -c` for JSON manipulation (no `jq` dependency).

```sh
peon-class          # Read: prints current class from .claude/settings.local.json
peon-class orc      # Write: creates/updates .claude/settings.local.json with env.PEON_SOUND_CLASS = "orc"
```

Valid values: `orc`, `human`, `undead`, `random`, `none`. Case-insensitive (lowercased by the function).

### settings.json Hook Registration

All hooks are in `~/.claude/settings.json` under `hooks`. Key configuration per hook:

| Hook Script | Event | Matcher | Timeout | Async |
|-------------|-------|---------|---------|-------|
| `tabtitle-hook.py` | `UserPromptSubmit` | (none) | 30s | yes |
| `tab-attention-hook.py` | `Notification` | (none) | 5s | yes |
| `tab-attention-hook.py` | `PostToolUse` | `AskUserQuestion` | 5s | yes |
| `plan-accept-hook.py` | `PostToolUse` | `ExitPlanMode` | 5s | yes |
| `tab-stop-question-hook.py` | `Stop` | (none) | 20s | yes |
| `session-sound-hook.py` | `SessionStart` | (none) | 5s | yes |

All hooks run async to avoid blocking the Claude Code UI. The `tab-attention-hook.py` is registered twice (two different events). The `tabtitle-hook.py` has a long 30s timeout to accommodate the Haiku subprocess call.
